# フィッティングの初心者向けTips

このプロジェクトでいう「フィッティング」は、主に `config.toml` の値を調整して、
クラスタリング結果がデータの見た目や実験の期待と合うように寄せていく作業です。
厳密にはピークフィットではなく、**クラスタリング条件の調整**に近いです。

## まず意識すること

- 1回で正解を当てようとしない  
  `DBSCAN` や **t-SNE / UMAP** は、少し値を変えるだけで見え方がかなり変わります。1つずつ変えて比較します。
- 何を揃えたいかを先に決める  
  「ピーク位置の違いを見たい」のか、「強度差も含めて分けたい」のかで設定が変わります。
- 見た目だけで判断しすぎない  
  **t-SNE や UMAP** は見やすいですが、図がきれいでも必ずしも物理的に意味のある分離とは限りません。

## おすすめの進め方

1. まず既定設定で実行する
2. ノイズが多すぎるか、全部つながるかを見る
3. `dbscan.eps` を先に調整する
4. 次に `dbscan.min_samples` を調整する
5. 必要なら **`embedding.method`**（t-SNE と UMAP の切り替え）や **`embedding.tsne` / `embedding.umap`**、`preprocess.intensity_normalization` を触る

一度に複数の値を変えると、どの変更が効いたのか分からなくなります。

## 各パラメータの見方

### `dbscan.eps`

- 一番効きやすいパラメータです
- 小さすぎると、ほとんどが `noise` になります
- 大きすぎると、全部が1クラスタにまとまりやすいです
- 迷ったら少しずつ増減します  
  例: `5 -> 8 -> 12 -> 18`

## `dbscan.min_samples`

- クラスタとして認める最小点数です
- 小さくすると小クラスタを拾いやすいです
- 大きくすると安定した塊だけ残りやすいです
- 点数が少ないデータでは、まず `2` か `3` から試すのが無難です

## `embedding.method`（右パネル）

- `tsne`  
  PCA 射影を入力とした **t-SNE 2D** を右パネルに表示します（既定でよく使う組み合わせ）。
- `umap`  
  同じく PCA 射影を入力とした **UMAP 2D** を右パネルに表示します。t-SNE より局所・大域のバランスを変えたいときに試します。
- `pca2d`  
  右パネルも **PC1 vs PC2** です（多様体は計算しません）。左と同じ平面を並べて見比べたい・計算を軽くしたいとき向きです。

**DBSCAN をかける空間**は別キー **`dbscan.clustering_space`** です。右パネルの算法と混同しないと整理しやすいです。

## `dbscan.clustering_space`

- `scaled`（既定）  
  **列標準化（`StandardScaler`）後の高次元特徴**でクラスタリングします。素直ですが、距離スケールはデータ依存で **`eps` の調整が重要**です。
- `pca`  
  **PCA 射影の全成分**（`pca.n_components` 次元、実際はデータ形状で上限）でクラスタリングします。
- `embedding`  
  **右パネルと同じ 2D 座標**でクラスタリングします（`embedding.method` が `tsne` / `umap` / `pca2d` のどれかに対応）。図の塊に近い結果を出しやすい一方、多様体のパラメータや乱数種にも引きずられます。
- `pca2d`  
  **PC1 と PC2 だけ**でクラスタリングします。PCA 平面上の分離を素直に見たいとき向きです（右パネルが t-SNE でも、DBSCAN はあくまで PC1–PC2 にかかります）。

初心者は、まず **`scaled`** で試し、うまく分かれないときに **`embedding`** や **`pca2d`** を検討すると進めやすいです。

### `pca2d` で 2 つに見えるのに 1 クラスタになる理由

- `pca2d` でも、分け方そのものは **DBSCAN** です
- つまり「PC1 のこの値で切る」のような明示的な境界線では分けていません
- `eps` 以内で点がつながっていると、見た目で少し離れていても 1 クラスタになります
- 間に数点の橋渡しがあると、2 群に見えても全体がつながることがあります

### そのときどうするか

- `dbscan.eps` を下げる
- `dbscan.min_samples` を上げる
- それでも 1 クラスタのままなら、DBSCAN がその分け方に向いていない可能性があります

「PC1 のしきい値で分けたい」「見た目どおりに 2 分割したい」という場合は、
DBSCAN ではなく別の方法が必要です。

### 表示とクラスタ空間の組み合わせの見方

- 左の図は **常に PC1 vs PC2** です
- 右の図は **`embedding.method`** で決まります（t-SNE / UMAP / 右も PCA）
- 例: `embedding.method = "tsne"` かつ `clustering_space = "pca2d"` のとき、**右は t-SNE、DBSCAN は PC1–PC2** です。図とクラスタの基準がずれるので、結果を解釈するときは **`clustering_space`** を意識してください
- `embedding.method = "pca2d"` のとき、右も PC1 vs PC2 なので幾何は左と同型ですが、**タイトルや軸ラベルで右パネルであること**は区別されます

## `embedding.tsne`（`perplexity` など）

- **t-SNE** の近傍スケールの目安です（`[embedding.tsne]`）
- 小さめだと細かい塊が出やすいです
- 大きめだと大まかな構造が出やすいです
- 点数が少ないデータでは、まず `3` から `10` くらいで試すのが現実的です
- `learning_rate` は多くの場合 **`"auto"` のまま**でよく、変えると収束の速さや形が変わります
- `max_iter` は **250 未満にできません**（scikit-learn の制約）。省略するとコード側の既定（大きめ）が使われます

## `embedding.umap`

- **`embedding.method = "umap"`** のときに効きます（設定は `[embedding.umap]`）
- `n_neighbors` を変えると局所構造と大域構造のバランスが変わります。小さめは局所、大きめは大域寄りのイメージです
- `min_dist` は点同士の詰まり具合に影響します。小さめでより密な塊、大きめでより広がった配置になりやすいです
- メトリックは通常 **`euclidean`** のままで十分なことが多いですが、データに応じて文献やチュートリアルを参照してください

## `pca.n_components`

- **t-SNE / UMAP の前**に、元の高次元特徴をいったん圧縮するための値です
- 大きすぎるとノイズまで持ち込みやすく、小さすぎると重要な違いを落としやすいです
- 初心者は、まず `10`, `20`, `30`, `50` あたりを比べると感覚をつかみやすいです
- サンプル数が少ない場合は、設定を大きくしても実際にはその数まで使われません  
  コード側で `min(n_components, サンプル数, 特徴数)` に制限されます

### PCA をどう考えればいいか

- PCA は「分かりやすい軸を作る前処理」です
- 左の PCA 図は、常に **第1主成分と第2主成分**を見る図です
- `n_components` を 2 より大きくしても、左の PCA 図そのものは基本的に変わりません
- **t-SNE / UMAP** の入力として使う場合は、細かいノイズを減らす役割もあります

### 迷ったときの目安

- **t-SNE / UMAP** の結果が毎回かなり不安定  
  `n_components` を少し下げる、または **`embedding.method = "umap"`** と **`random_state` 固定**で比較を試す
- 分離が弱くて、どれも似た塊に見える  
  `n_components` を少し上げる
- まず様子を見たい  
  `20` 前後から始める
- 特徴数がかなり多い  
  `30` から `50` を候補にする

## `preprocess.intensity_normalization`

- `l2`  
  パターンの形を比較しやすいです。まずはこれで十分です。
- `max`  
  最大ピークを揃えたいときに使います。
- `none`  
  強度差そのものを残したいときに使いますが、強度の大きい試料に引っ張られやすいです。

## `grid.exclude_ranges`

- 特定の 2theta 範囲を特徴量から除外できます
- 薄膜試料で基板ピークが強すぎるときに有効です
- 補間したあと、**クラスタリング用特徴量に対してだけ**除外されます
- まずは「基板の既知ピークの前後を少し広めに切る」くらいから始めるのが無難です
- 例:

```toml
[grid]
theta_min = 20.0
theta_max = 80.0
n_points = 601
exclude_ranges = [[26.4, 27.2], [54.1, 55.0]]
```

### 使い方のコツ

- いきなり広く切りすぎない
- 膜ピークまで消していないか確認する
- 1本ずつレンジを追加して、クラスタの変化を見る
- 「基板ピークを消したら分離が良くなるか」を先に見る
- 下段の XRD プロファイル図では、除外した範囲も残したまま灰色帯で見えるので、切りすぎ確認に使えます

## `visualize.xrd_min_panel_height_px`

- 各 XRD 図の高さの基準値です
- 実際の描画時には、この値を下限として **さらに約 1.5 倍**した高さが使われます
- 大きくすると見やすくなりますが、HTML 全体は縦に長くなります

例:

```toml
[visualize]
xrd_min_panel_height_px = 320
```

## XRD 表示の使い方

- XRD は 1 枚の subplot ではなく、**All profiles** と **各クラスタ別の複数グラフ**に分かれています
- デスクトップでは **2 列レイアウト**、狭い画面では **1 列**になります
- 各グラフの legend は独立しているので、あるクラスタ図で消した系列は他のクラスタ図に影響しません
- 上部フォームから、**全 XRD 図の X/Y 範囲**と **Y 軸の linear/log** を一括変更できます
- グレー帯は `grid.exclude_ranges` で、クラスタリングでは無視した 2theta 範囲を示しています

## ウェハーマップの見方

- 埋め込み図の横に、**ファイル名の X/Y 座標**を使ったウェハーマップ風パネルが出ます
- 色はクラスタ色と対応しているので、位置依存の傾向があるかをざっくり確認できます
- 位置はファイル名由来なので、座標の命名がずれていると正しく見えません

## 右サイドバーの見方

- 右側には `Analysis metadata` と `Files by cluster` のカードが出ます
- `Files by cluster` を見ると、散布図や XRD で気になったクラスタにどのファイルが入っているかをすぐ追えます
- 情報量が多いときは、散布図や XRD と見比べながらサイドバー内だけスクロールすると確認しやすいです

## 迷ったときの判断基準

- ノイズが多すぎる  
  `eps` を上げる
- 全部が1クラスタになる  
  `eps` を下げる
- 細かく割れすぎる  
  `min_samples` を上げる
- 図では分かれて見えるのにクラスタが出ない  
  `clustering_space = "embedding"` を試す（右パネルと同じ 2D で DBSCAN）
- 高次元の距離感が合わずノイズだらけ  
  `clustering_space = "pca"` や **`pca2d`** を試す
- t-SNE が不安定で比較しづらい  
  **`embedding.method = "umap"`** や **`pca2d`** を試す、または `clustering_space = "pca2d"` で **PC 平面上だけ**でクラスタを決める
- `pca2d` で 2 群に見えるのに 1 クラスタになる  
  `eps` を下げるか `min_samples` を上げる
- **t-SNE** の見え方が不安定  
  `pca.n_components` を下げてみる、または **UMAP** に切り替えて `random_state` を固定する
- 基板ピークに引っ張られていそう  
  `grid.exclude_ranges` でその 2theta 範囲を外す
- 強度差よりピーク形状で分けたい  
  `intensity_normalization = "l2"` を使う

## 初心者向けの実践ルール

- 変更前の `config.toml` を保存しておく
- 実行ごとに結果の HTML を別名で残す
- 「なぜこの設定にしたか」を1行メモする
- 良さそうな結果が出ても、近い値でもう1回だけ確認する
- XRD フォームで全体の軸範囲を揃えてから、各図の legend で見たい系列だけ残す
- ウェハーマップも合わせて見て、クラスタが空間的に連続しているか確認する

## 最後に

良いフィッティングは、「きれいな図」ではなく「再実行しても納得できる設定」です。
まずは大まかに当たりを付けてから、少しずつ絞るのが一番安定します。
